Technology:
JTextField in Java
Author:
Jonathon Thach
Summary:
When designing a GUI, there are plently of classes to pick from that will enhance the look and use of your website or application. For many developers, they require user input, then they perform an action, and return the results back to the user. JTextFields are one of those classes that allow for such a thing. This TechNote will descibe how JTextFields work with an example using ActionListeners, JButtons, and JLabels.
Description:

Adding a text field to a GUI is as simple as calling the constructor of the JTextFields class. 


JTextField textField = new JTextField (“Please type in me!”);
GUI.add(textField);

In this example, we created a text field with a text already in it. It looks like this after running the code. You can click on the TextField, delete the text inside and type whatever you would like. Here is what it looks like.

We can remove the text from the text field to make the box empty, but we would need to input an int to determine the width of the text field afterwards. Afterwards we will add a Jlabel to our GUI. 


// Width of 30 columns. 
JTextField textField = new JTextField (30);

JLabel is another constructor but it is used to display a string, an image, or both. But in this example, I will be just displaying a string. 


JLabel labelText = new JLabel("This text will change after you press the Enter key.");
GUI.add(labelText);

This code calls the JLabel constructor and sets the labelText to The string inside. It looks something like this.

Now an ActionListener is a listener interface that receives data after an action event occurs. Now there are a few ways of coding an ActionListener such as using a button that activates the action. But for this example, we will make the Enter key on our keyboard our ActionListener. For that to happen the ActionListener will need something call actionPerformed to make that happen The code for something like that would look like this. 


textField.addActionListener(new ActionListener(){

    // Enter
    public void actionPerformed(ActionEvent e){

        labelText.setText(textField.getText());

    }
});

Luckily, the default ActionEvent will be Enter if the event handler is for textField. But it could be for a button if we used a JButton. Putting the entire example together we will have a simple program that will have a text field, a label, and an action listener that will change the label depending on what Is typed in the text field. The more complete code would similar to this. 


final JFrame frame = new JFrame("Example");

JTextField textField  = new JTextField(30);

JLabel labelText = new JLabel("This text will change after you press the Enter key.");


textField.addActionListener(new ActionListener(){

    // Enter
    public void actionPerformed(ActionEvent e){

        labelText.setText(textField.getText());

    }
});
